## 🎯 지금 해결할 작업: 위험점수 정합성 (코드 ✅ 완료 / 보고서 ⏳ 남음)

### 상태
이 패키지의 `run_phase2.py`에는 **이미 표준 위험점수 산식이 반영되어 있고, csv도 갱신 완료**다. 남은 것은 **보고서 초안의 수치를 코드 결과와 맞추는 것**뿐이다.

### 적용된 표준 산식 (run_phase2.py에 반영됨)
```python
RISK_WEIGHTS = {'MI': 0.40, 'CPC': 0.25, 'LRR': 0.20, 'TREND': 0.15}
def _minmax(s): return (s - s.min()) / (s.max() - s.min())
위험점수 = (_minmax(-MI)*0.40 + _minmax(-CPC)*0.25
            + _minmax(-LRR)*0.20 + _minmax(-TREND)*0.15)
```

### 남은 일 (보고서 동기화)
1. `보고서_초안_v1.md` 4.1 표의 위험점수 열을 위 표 수치로 교체.
   (초안은 하동0.829·산청0.759·합천0.754·의령0.749·함양0.712 였음 → 위 정답으로)
2. 5장 처방 순서가 새 순위(하동>함양>합천>남해>의령)와 어긋나면 조정.
   ※ 단, 초안의 "처방은 유형군 단위" 원칙은 유지 가능 — 순위는 우선순위 참고용.
3. 보고서 3장 또는 4.1에 위험점수 산식을 명시적으로 공개(아래 문장 삽입):
   > "위험점수 = 0.40·MI' + 0.25·CPC' + 0.20·LRR' + 0.15·TREND'
   >  (각 지표 13개 지역 min-max 정규화, 낮을수록 위험인 지표는 부호 반전).
   >  가중치는 핵심 가설인 소비-유동 불일치(MI)에 최대 비중 부여."

## 핵심 지표 정의 (9종)

| 지표 | 정의 | 해석 |
|------|------|------|
| **MI** ★ | log(연간매출/연평균 생활인구) − 13곳 평균 | 음수 = 소비전환 취약 |
| **CPC** ★ | 연간매출/생활인구/12 | 1인당 월 소비 흡수력 |
| **STI** ★ | 성수기(6~8월)/비수기(1~3월) − 1 | 높음 = 관광 의존 |
| LRR | 경남 유입 매출/전체 | 낮음 = 외부 소비자 의존 |
| CDI | 업종 매출 Shannon Entropy | 낮음 = 업종 집중 |
| NAR | 18~23시 매출/전체 | 낮음 = 야간 약함 |
| AGI | 60대+ 매출 비중 | 높음 = 고령 의존 |
| YR | 20~30대 매출 비중 | 낮음 = 청년 이탈 |
| TREND | 월별 매출 회귀 기울기/평균 | 음수 = 감소세 |

★ = 생활인구 결합으로 가능해진 핵심 지표

> ⚠️ LRR 해석 주의: 낮은 LRR은 "자금 유출"이 **아니라** 해당 지역 매출 중 경남 외 소비자 비중이 높다는 의미(외부유입 의존). 보고서 초안은 이미 올바르게 수정됨. 코드 주석이나 옛 문서에 "출혈성"이라 적힌 게 있으면 정정할 것.

## 침체 4유형 (실제 분류 결과)

- **소비전환 취약형** (산청·의령·합천): MI < -0.6, CPC < 17,000
- **외부의존+소비전환 취약형** (하동·함양): LRR 낮음 + MI 음수 + TREND ≤ 0
- **경계형** (남해·고성·창녕·함안)
- **상대적 양호형** (통영·사천·거창·밀양): MI 양수, CPC 높음

> ⚠️ 시각화 PNG의 범례는 옛 용어("출혈+소화 복합형" 등)일 수 있음.
> 보고서 용어("외부의존+소비전환 취약" 등)와 통일하려면 `run_phase2.py`의
> `type_colors`와 `name_cluster_v2()` 함수의 유형명을 수정 후 재생성.
